1.概述

HashMap是基于哈希表的Map接口实现，是以Key-Value形式而存在的，HashMap的实现不是同步的，意味它不是线程安全的，此外它的Key Value 都可以为null，它没有存储数据是无序的。

2.类图

3.成员属性

3.1 serialVersionUID

private static final long serialVersionUID = 362498820763181265L;

序列化版本号

3.2 DEFAULT_INITIAL_CAPACITY

static final int DEFAULT_INITIAL_CAPACITY = 1 << 4;

默认的初始化容量，必须是2的幂

思考：为什么必须是2的幂呢，因为求出来的hash值需要散列到数组的某一下标，这个操作可以使用取模运算,但是效率比较低，只有数组为2的幂的时候才能使用按位与的操作代替取模运算。

3.3 MAXIMUM_CAPACITY

static final int MAXIMUM_CAPACITY = 1 << 30;

允许的最大容量

3.4 DEFAULT_LOAD_FACTOR

static final float DEFAULT_LOAD_FACTOR = 0.75f;

默认复杂因子: 是用来衡量HashMap稀疏程度，影响到Hash冲突的概率，如果太大数组就会比较的拥挤，影响操作的时间复杂度，如果太小就会比较的稀疏，浪费很多的内存。而0.75是官方给出的一个在时间和空间上的一个比较平衡的值。

3.5 TREEIFY_THRESHOLD

static final int TREEIFY_THRESHOLD = 8;

判断是否需要树化的阈值

思考：为什么是链表节点的个数大于8时进行树化，1. 因为树节点的大小是普通节点的2倍，得需要足够得节点才能树化，2. 在用户的实现的hashCode方法散列性比较好的时候，链表节点个数为8的概率服从泊松分布，为很低的概率，几乎为不可能事件，如果hashCode方法实现的不好,JDK也没有办法阻止用户，所以引用红黑树来改善链表节点过多的情况

3.6 UNTREEIFY_THRESHOLD

static final int UNTREEIFY_THRESHOLD = 6;

判读是否需要转成链表的阈值

思考：为什么是6进行链表化：应该要比8小，避免频繁的转化过程

3.7 MIN_TREEIFY_CAPACITY

static final int MIN_TREEIFY_CAPACITY = 64;

判断是否需要树化的数组的长度值

思考：为什么要满足>=64才进行树化呢：还未有很好的见解

3.8 table

transient Node<K,V>[] table;

存储节点的数组

3.9 entrySet

transient Set<Map.Entry<K,V>> entrySet;

调用entrySet()方法后的缓存

3.10 size

transient int size;

map中的键值对的数量，并不等于数组的长度

3.11 modCount

transient int modCount;

map的修改结构的次数

3.12 threshold

int threshold;

判断是否需要扩容的阈值：计算方式 = loadfactor * table.length

3.13 loadFactor

final float loadFactor;

真正存储负载因子变量

4. 内部类

4.1 Node

static class Node<K,V> implements Map.Entry<K,V> {
    //存储hash值
    final int hash;
    //存储键值
    final K key;
    //存储值
    V value;
    //存储下一节点地址
    Node<K,V> next;
    //构造函数
    Node(int hash, K key, V value, Node<K,V> next) {
        this.hash = hash;
        this.key = key;
        this.value = value;
        this.next = next;
    }
    //获取键
    public final K getKey()  { return key; }
    //获取值
    public final V getValue()      { return value; }
    public final String toString() { return key + "=" + value; }
    //计算出hashCode
    public final int hashCode() {
        //调用key和value的hashCode方法做异或运算
        return Objects.hashCode(key) ^ Objects.hashCode(value);
    }
    //设置值
    public final V setValue(V newValue) {
        V oldValue = value;
        value = newValue;
        return oldValue;
    }
    //判断Node是否相同
    public final boolean equals(Object o) {
        //如果传进来的是自身
        if (o == this)
            return true;
        //如果是其实现类或对象
        if (o instanceof Map.Entry) {
            Map.Entry<?,?> e = (Map.Entry<?,?>)o;
            // key 和 value 都需要满足是本身的情况 || 满足自身的equals方法
            if (Objects.equals(key, e.getKey()) &&
                Objects.equals(value, e.getValue()))
                return true;
        }
        return false;
    }
}

5. 构造方法

5.1 HashMap()

public HashMap() {
    this.loadFactor = DEFAULT_LOAD_FACTOR; 
}

初始化默认的负载因子,并未指定table数组的大小

5.2 HashMap(int initialCapacity)

public HashMap(int initialCapacity) {
    this(initialCapacity, DEFAULT_LOAD_FACTOR);
}

调用了另一个构造函数，不过指定了负载因子为默认的负载因子

5.3 HashMap(int initialCapacity, float loadFactor)

public HashMap(int initialCapacity, float loadFactor) {
    //如果传入的大小小于0抛出异常
    if (initialCapacity < 0)
        throw new IllegalArgumentException("Illegal initial capacity: " +
                                           initialCapacity);
    //如果传入的大小大于最大容量 就赋值为最大容量
    if (initialCapacity > MAXIMUM_CAPACITY)
        initialCapacity = MAXIMUM_CAPACITY;
    //判断负载因子loadFactor是否小于等于0 || 是否是一个非数值
    if (loadFactor <= 0 || Float.isNaN(loadFactor))
        throw new IllegalArgumentException("Illegal load factor: " +
                                           loadFactor);
    //将指定负载因子赋值到HashMap成员变量的负载因子loadFactor
    this.loadFactor = loadFactor;
    //将方法计算出的值赋值给扩容的阈值
    this.threshold = tableSizeFor(initialCapacity);
}
//返回比指定容量大的最小的2的n次幂
static final int tableSizeFor(int cap) {
     int n = cap - 1;
     n |= n >>> 1;
     n |= n >>> 2;
     n |= n >>> 4;
     n |= n >>> 8;
     n |= n >>> 16;
     return (n < 0) ? 1 : (n >= MAXIMUM_CAPACITY) ? MAXIMUM_CAPACITY : n + 1;
}

思考：此处的table为什么一定要是返回比指定容量大的最小的2的n次幂呢 ，用户可能不知道HashMap内部的实现原理，可能会出现乱指定容量的情况，为了避免降低HashMap的性能，因为put采取的按位与的方法需要数组的长度是2的n次幂。

5.4 HashMap(Map<? extends K, ? extends V> m)

public HashMap(Map<? extends K, ? extends V> m) {
    //将loadFactor赋值为默认值
    this.loadFactor = DEFAULT_LOAD_FACTOR;
    putMapEntries(m, false);
}

final void putMapEntries(Map<? extends K, ? extends V> m, boolean evict) {
    //将s赋值为
    int s = m.size();
    if (s > 0) {
        if (table == null) { // pre-size
            float ft = ((float)s / loadFactor) + 1.0F;
            int t = ((ft < (float)MAXIMUM_CAPACITY) ?
                     (int)ft : MAXIMUM_CAPACITY);
            if (t > threshold)
                threshold = tableSizeFor(t);
        }
        else if (s > threshold)
            resize();
        for (Map.Entry<? extends K, ? extends V> e : m.entrySet()) {
            K key = e.getKey();
            V value = e.getValue();
            putVal(hash(key), key, value, false, evict);
        }
    }
}

6.成员方法

6.1 哈希函数 - hash(Object key)

static final int hash(Object key) {
    int h;
    //如果key等于null 值为 0    用key的hahsCode与自身的低16位做异或运算
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

思考：为什么要这样做呢：这段代码叫做扰动函数：假设用户实现的hashCode方法很好，并没有很多的Hash冲突，由于put方法采用是取数组长度-1的那几位与hash值做按位与的运算，那么发生Hash冲突的概率会很高，加上如果hashCode实现的不是很好的情况下，情况会更加的糟糕，所以取hashCode的高16位与低16位做异或运算，混合hashCode的高位和低位，以此来加大低位的随机性。

6.2 添加单个元素 put(K key, V value)

public V put(K key, V value) {
    return putVal(hash(key), key, value, false, true);
}

final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
               boolean evict) {
    //创建tab数组，创建Node指针p ,定义2个整型变量  
    Node<K,V>[] tab; Node<K,V> p; int n, i;
    //如果条件成立，说明数组未进行初始化是第一次进入put方法
    if ((tab = table) == null || (n = tab.length) == 0)
        //调用扩容的方法将数组长度赋值给n 第一次长度等于16
        n = (tab = resize()).length;
    /**
    * 1. 用来将hash值散列到数组中的操作
    * 2. p指向数组某个下标中的节点 如果为null说明没有节点可以创建新节点。
    * 3. 计算索引的方式为使用数组的长度-1和用key计算出来的hash值，做按位与的运算相当于取模运算
    * 4. 其他方法 1. 平方取中法 2. 伪随机数法  3. 取余数法  效率都比较低
    */
    if ((p = tab[i = (n - 1) & hash]) == null)
        tab[i] = newNode(hash, key, value, null);
    //否则就是有节点,可能发生了Hash碰撞
    else {
        //定义Node指针e K型的变量
        Node<K,V> e; K k;
        /** 
         *   1. 判断新节点与旧节点hash值是否相同
         *   2. 前面成立 判断新key和旧key地址值是否相同，是就为同一key
         *   3. 或者说明key的地址不相同,判断key的内容是否相等，是就为同一key
         */
        if (p.hash == hash && ((k = p.key) == key || (key != null && key.equals(k))))
            //e指向旧节点
            e = p;
        //判断旧节点是否为红黑树节点
        else if (p instanceof TreeNode)
            //e指向null 将新节点放入红黑树中
            e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
        //不是红黑树节点
        else {
            //用循环是因为链表中不仅仅只有一个节点: 1. 需要遍历到最后一个节点插入 2. 在遍历的过程中判断是否有重复的key
            for (int binCount = 0; ; ++binCount) {
                //说明已经移动到最后一个节点
                if ((e = p.next) == null) {
                    //插入到后面
                    p.next = newNode(hash, key, value, null);
                    /**
                    * 1. 添加节点完成之后 判断是否需要转化成红黑树
                    * 2. 数组是从0开的循环的，加上数组中的的第一个一元素，元素个数是9
                    */
                    if (binCount >= TREEIFY_THRESHOLD - 1)
                        treeifyBin(tab, hash);
                    //树化后退出
                    break;
                }
                //判断是否重复的key
                if (e.hash == hash &&
                    ((k = e.key) == key || (key != null && key.equals(k))))
                    //退出进行值的覆盖
                    break;
                //移动到链表下一节点
                p = e;
            }
        }
        //进行值的替换
        if (e != null) {
            //旧值
            V oldValue = e.value;

            if (!onlyIfAbsent || oldValue == null)
                e.value = value;
            afterNodeAccess(e);
            return oldValue;
        }
    }
    ++modCount;
    //如果添加一个元素之后键值对的数量大于扩容阈值就进行扩容
    if (++size > threshold)
        resize();
    afterNodeInsertion(evict);
    return null;
}

6.3 扩容 resize()

final Node<K,V>[] resize() {
    //得到当前数组
    Node<K,V>[] oldTab = table;
    //得到当前数组容量
    int oldCap = (oldTab == null) ? 0 : oldTab.length;
    //得到当前数组的扩容阈值
    int oldThr = threshold;
    //定义新的容量，新扩容阈值
    int newCap, newThr = 0;
    //如果老数组的容量大于0
    if (oldCap > 0) {
        //如果老数组的容量大于等于最大容量  就不再进行扩容了
        if (oldCap >= MAXIMUM_CAPACITY) {
            //修改扩容阈值为整型的最大值
            threshold = Integer.MAX_VALUE;
            return oldTab;
        }
        //将数组扩容为原来的2倍 要满足小于最大容量 大于等于默认初始化容量
        else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY &&
                 oldCap >= DEFAULT_INITIAL_CAPACITY)
            //扩容阈值增大2倍
            newThr = oldThr << 1; 
    }
    //如果老阈值大于0
    else if (oldThr > 0) 
        //将老阈值赋值给新数组容量
        newCap = oldThr;
    else {
        //直接使用默认初始容量
        newCap = DEFAULT_INITIAL_CAPACITY;
        //计算新的扩容阈值
        newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY);
    }
    //不是第一次进入方法  
    if (newThr == 0) {
        //计算出新的扩容阈值
        float ft = (float)newCap * loadFactor;
        //新容量和新阈值必须满足小于最大值  满足赋值给新阈值属性，否则赋值一个整型最大值
        newThr = (newCap < MAXIMUM_CAPACITY && ft < (float)MAXIMUM_CAPACITY ?
                  (int)ft : Integer.MAX_VALUE);
    }
    //将newThr赋值给threshold属性
    threshold = newThr;
    @SuppressWarnings({"rawtypes","unchecked"})
        //创建新数组
        Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];
    //将table指向新数组
    table = newTab;
    //如果不是第一次进入方法 就进行数组的搬运
    if (oldTab != null) {
        //遍历每一个桶
        for (int j = 0; j < oldCap; ++j) {
            //定义Node指针
            Node<K,V> e;
            //桶口有节点
            if ((e = oldTab[j]) != null) {
                //将桶口的引用置为null
                oldTab[j] = null;
                //说明桶中只有一个节点
                if (e.next == null)
                    //直接用原来的hash值与新数组长度-1按位与散列到新数组中去
                    newTab[e.hash & (newCap - 1)] = e;
                //说明不止一个节点 判断是否是红黑树节点
                else if (e instanceof TreeNode)
                    //调用相关红黑树方法进行处理
                    ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);
                //将原来的数据进行散列的搬移
                else { 
                    /**
                    * 处理重新散列过后在新数组的位置等于原数组位置的情况
                    *  loHead指向链表的首节点  loTail指向链表的尾节点
                    */  
                    Node<K,V> loHead = null, loTail = null;

                    /**
                    *  处理重新散列过后在新数组中的位置等于原数组位置+原数组长度的情况
                    *      hiHead指向链表的首节点     hiTail指向链表的尾节点
                    */
                    Node<K,V> hiHead = null, hiTail = null;
                    //定义了一个Node型的next指针
                    Node<K,V> next;

                    do {
                        //当前节点的下一节点
                        next = e.next;
                        //原来的hash与原有的长度按位与运算  结果==0 说明散列到了原数组位置
                        if ((e.hash & oldCap) == 0) {
                            //将所有散列到原数组位置的节点形成链表
                            if (loTail == null)
                                loHead = e;
                            else
                                loTail.next = e;
                            loTail = e;
                        }

                        else {
                            //将所有散列到原数组位置+原数组长度的节点形成链表
                            if (hiTail == null)
                                hiHead = e;
                            else
                                hiTail.next = e;
                            hiTail = e;
                        }

                    } while ((e = next) != null);

                    /**
                    * 直接将重新散列后不需要移动位置的链表的头，放入新数组中的位置就是原来的位置，
                    * 并将尾节点的next指向null
                    */
                    if (loTail != null) {
                        loTail.next = null;
                        newTab[j] = loHead;
                    }

                    /** 
                    * 直接将重新散列后需要移动位置的链表的头，放入新数组中的位置就是原来的位置+原数组长度，
                    * 并将尾节点的next指向null
                    */
                    if (hiTail != null) {
                        hiTail.next = null;
                        newTab[j + oldCap] = hiHead;
                    }

                }
            }
        }
    }
    return newTab;
}

说明: 进行数据的搬移的时候：新节点的位置取决于扩容后那一位的值，因为数组被设计成2的n次幂,每次扩容为原来的2倍。如果为1：那么节点在新数组的位置为：原来的数组的位置+原数组长度，如果为0那么节点在新数组的位置为：原来数组的位置

6.4 移除单个元素 - remove(Object key)

public V remove(Object key) {
    Node<K,V> e;
    return (e = removeNode(hash(key), key, null, false, true)) == null ?
        null : e.value;
}

final Node<K,V> removeNode(int hash, Object key, Object value,
                           boolean matchValue, boolean movable) {
    Node<K,V>[] tab; Node<K,V> p; int n, index;
    if ((tab = table) != null && (n = tab.length) > 0 &&
        (p = tab[index = (n - 1) & hash]) != null) {
        Node<K,V> node = null, e; K k; V v;
        if (p.hash == hash &&
            ((k = p.key) == key || (key != null && key.equals(k))))
            node = p;
        else if ((e = p.next) != null) {
            if (p instanceof TreeNode)
                node = ((TreeNode<K,V>)p).getTreeNode(hash, key);
            else {
                do {
                    if (e.hash == hash &&
                        ((k = e.key) == key ||
                         (key != null && key.equals(k)))) {
                        node = e;
                        break;
                    }
                    p = e;
                } while ((e = e.next) != null);
            }
        }
        if (node != null && (!matchValue || (v = node.value) == value ||
                             (value != null && value.equals(v)))) {
            if (node instanceof TreeNode)
                ((TreeNode<K,V>)node).removeTreeNode(this, tab, movable);
            else if (node == p)
                tab[index] = node.next;
            else
                p.next = node.next;
            ++modCount;
            --size;
            afterNodeRemoval(node);
            return node;
        }
    }
    return null;
}
