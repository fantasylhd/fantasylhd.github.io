day12static和final和内部类

1.1 final 关键字

1.1.1 final关键字使用

可以修饰：

  • 类：

格式: public final class 类名

特点: 被final修饰的类不能被继承

  • 方法：

格式:
修饰符 final 返回值类型 方法名(参数){
方法体
返回值
}

特点:被final修饰的方法不能被重写

  • 局部变量：

格式:
final 数据类型 变量名 = 值

特点:
被final修饰的局部变量一旦被初始化,那么不能被二次赋值

注意： 对象也属于局部变量：

格式:
final 类名 对象名 = new 类名();

特点:
被final修饰的对象名,地址值不能改变,但是对象中的属性可以改变

  • 成员变量：

格式:
final 数据类型 变量名

特点:
被final修饰的成员变量不能被二次赋值

注意:
1.要不就手动赋值
2.要不就利用所有的构造初始化

  • 对象

格式:
final 类名 对象名 = new 类名();

特点:
被final修饰的对象名,地址值不能改变,但是对象中的属性可以改变

总结：就是不可修改

1.2 static

1.2.1 static关键字使用

  • Static的介绍

可以修饰：

  • 类:

java里面static一般用来修饰成员变量或函数。但有一种特殊用法是用static修饰内部类，普通类是不允许声明为静态的，只有内部类才可以。下面看看如何使用。

  • 方法：

可以直接通过类名来进行调用。

  • 变量：

被static修饰的成员变量叫做静态变量，也叫做类变量，说明这个变量是属于这个类的，而不是属于是对象，没有被static修饰的成员变量叫做实例变量，说明这个变量是属于某个具体的对象的。

  • 代码块：

注意:
1.优先于构造方法执行
2.只执行一次

使用场景:
当我们初始化变量的时候,只要求我们初始化一次,可用供后面的使用

1.2.2 静态方法的访问特点

  1.静态的不能直接访问非静态的
    静态的属于类,非静态的属于对象,类一旦加载,那么静态的就会跟着类直接进内存,是优先于对象存在的

    使用:创建对象,再调用非静态的方法

  2.静态的能直接访问静态

     同类:
        直接调用
        创建对象调用
        类名直接调用
     不同类:
        创建对象调用
        类名直接调用

   3.非静态的能直接访问静态的
      同类:
         直接调用
         创建对象调用
         类名直接调用

      不同类:
          创建对象调用
          类名直接调用
  • Static内存图

总结 ：方便在没有创建对象的情况下进行调用(方法/变量)。

1.4 内部类

1.4.1 内部类的定义

所谓内部类就是在一个类内部进行其他类结构的嵌套操作。

1.4.2 内部类的分类

  • 成员内部类

定义和使用：

需要注意的是， 当成员内部类拥有和外部类同名的成员变量或这方法时， 默认情况下访问的是内部类的成员， 如要访问外部类的同名成员， 需要使用以下形式：

外部类.this.成员变量

外部类.this.成员方法

内部类对象创方式：外部类.内部类 对象名 = new 外部类().new 内部类()

注意:
内部类可以直接使用外部类的成员 外部类不能直接使用内部类的成员

  • 静态内部类

定义和使用：

内部类对象创方式： 外部类.内部类 内部类对象 = new 外部类.内部类();

静态内部类也是定义在另一个类里面的类，只不过在类前加上了static。静态内部类是不需要依赖于外部类的，与静态成员变量类似。

  • 方法内部类

定义和使用：

方法内部类不允许使用访问权限修饰符（public、private、protected）均不允许。

局部内部类是定义在一个方法或作用域中的类，它的访问权限仅限于方法内或作用域内。

  • 匿名内部类

定义和使用：

匿名内部类就是一个没有名字的方法内部类，因此特点和方法与方法内部类完全一致，除此之外，还有自己的特点：

1.匿名内部类必须继承一个抽象类或者实现一个接口。

2.匿名内部类没有类名，因此没有构造方法。
