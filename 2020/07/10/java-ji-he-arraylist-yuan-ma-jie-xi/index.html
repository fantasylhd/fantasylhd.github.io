1.概述

ArrayList是List接口的可调整大小的数组实现，实现了数组的一些基本操作，允许 put null 值，会自动扩容；size、isEmpty、get、set、add 等方法时间复杂度都是 O (1)。

2.ArrayList的类图

ArrayList继承的抽象类和实现的接口如下图所示：

实现了4个接口，分别是：

2.1 List

实现了集合的一些常用的方法。

2.2 Serializable（标记性接口）

表示 ArrayList 支持序列化的功能。

2.3 Cloneable （标记性接口）

表示 ArrayList 支持克隆。

2.4 RandomAccess（标记性接口)

表示 ArrayList 支持快速的随机访问。

2.5 AbstractList抽象类

AbstractList类实现了List接口的骨架实现，为下面的子类减轻了负担，提高了代码的复用性，不过实际上，在下面中我们会看到，ArrayList 大量重写了 AbstractList 提供的方法实现。所以，AbstractList 对于 ArrayList 意义不大，更多的是 AbstractList 其它子类享受了这个福利。

3.成员变量

3.1 serialVersionUID

通过此id来验证版本的一致性。

3.2 DEFAULT_CAPACITY

表示数组的初始大小，默认是 10。

3.3 EMPTY_ELEMENTDATA

用于空对象的空数组对象。

3.4 DEFAULTCAPACITY_EMPTY_ELEMENTDATA

用于默认构造下的空对象的空数组对象

思考：EMPTY_ELEMENTDATA 和DEFAULTCAPACITY_EMPTY_ELEMENTDATA的区别：EMPTY_ELEMENTDATA用于除了空参构造之外的其余两个构造方法，用户可能想构造一个空数组的需求，DEFAULTCAPACITY_EMPTY_ELEMENTDATA用于默认情况下的空参构造方法，如果发生了扩容，默认情况下空参构造会从10开始扩容，而其他的构造在未指定大小的情况下，会从0开始扩容。

3.5 elementData

存储ArrayList元素数组缓冲区。

3.6 size

表示当前数组的大小 包含元素的数量。

4.构造方法

4.1 ArrayList()

   public ArrayList() {
        //将elementDate指向一个空的数组
        this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;
    }

思考：此处并未初始大小为10的数组，构造一个ArrayList对象，空参构造应该指向一个空数组,节省内存，一些场景有可能只需要创建ArrayList对象。

4.2 ArrayList(int initialCapacity)

  public ArrayList(int initialCapacity) {
        //如果指定的数字大于0，按指定的数字创建数组
        if (initialCapacity > 0) {
            this.elementData = new Object[initialCapacity];
        } else if (initialCapacity == 0) {
            //如果指定的数字等于0，将elementDate指向一个空数组
            this.elementData = EMPTY_ELEMENTDATA;
        } else {
            //以上都不满足则抛出异常
            throw new IllegalArgumentException("Illegal Capacity: "+initialCapacity);
        }
    }

思考：用户指定初始化容量为0应该指向一个空数组。

4.3 ArrayList(Collection<? extends E> c)


   public ArrayList(Collection<? extends E> c) {
        //将elementData指向一个集合对象转换成数组对象
        elementData = c.toArray();
        //如果转换过的数组对象的长度不等于0
        if ((size = elementData.length) != 0) {
            // c.toArray might (incorrectly) not return Object[] (see 6260652)
            //如果转换过对象的类的类型不等于对象数组类型
            if (elementData.getClass() != Object[].class)
                //elementData指向新的类型数组
                elementData = Arrays.copyOf(elementData, size, Object[].class);
        } else {
            //其他情况将elementData指向一个空数组
            this.elementData = EMPTY_ELEMENTDATA;
        }
    }
    //转换对象数组的方法
    public Object[] toArray() {
        //调用copyof方法返回对象数组
        return Arrays.copyOf(elementData, size);
    }

    public static <T> T[] copyOf(T[] original, int newLength) {
        //调用另一个copy方法，将数组的类型传入进去
        return (T[]) copyOf(original, newLength, original.getClass());
    }

    public static <T,U> T[] copyOf(U[] original, int newLength, Class<? extends T[]> newType) {
        //如果是对象类型创建对象类型数组，否则创建newType类型
        T[] copy = ((Object)newType == (Object)Object[].class) ? (T[]) new Object[newLength]
        : (T[]) Array.newInstance(newType.getComponentType(), newLength);
        //进行数组内容的搬移
        System.arraycopy(original, 0, copy, 0, Math.min(original.length, newLength));
        return copy;
    }

5. 成员方法

5.1 ArrayList的扩容机制

   //计算容量
   private static int calculateCapacity(Object[] elementData, int minCapacity) {  
        //如果是调用的空参构造，返回默认容量10
        if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
            return Math.max(DEFAULT_CAPACITY, minCapacity);
        }
        //否则的话就返回minCapacity
        return minCapacity;
        }

    //保证内部数组容量
    //minCapacity为 size+1 ,一个集合必须能够容纳size+1个元素,不够则会扩容
    private void ensureCapacityInternal(int minCapacity) {
        ensureExplicitCapacity(calculateCapacity(elementData, minCapacity));
    }

    //保证容量足够
    private void ensureExplicitCapacity(int minCapacity) {
        //记录数组被修改的次数
        modCount++;
        //如果size+1 > 目前数组的长度 则进行扩容
        if (minCapacity - elementData.length > 0)
            grow(minCapacity);
    }

    //进行扩容
    private void grow(int minCapacity) {
        //旧的容量
        int oldCapacity = elementData.length;
        //新的容量 = 旧的容量+旧容量/2
        int newCapacity = oldCapacity + (oldCapacity >> 1);
        //如果新容量小于size+1 则将size+1赋值给新容量 
        if (newCapacity - minCapacity < 0)
            newCapacity = minCapacity;
        //如果新的容量 > 整型的最大值-8 
        if (newCapacity - MAX_ARRAY_SIZE > 0)
            //新的容量 = 计算的最大容量
            newCapacity = hugeCapacity(minCapacity);
        //将原数组的内容移动到新数组去，又将elementData指向新的数组
        elementData = Arrays.copyOf(elementData, newCapacity);
    }

    //最大容量
    private static int hugeCapacity(int minCapacity) {
        //如果小于0，则表明发生了溢出
        if (minCapacity < 0) // overflow
            throw new OutOfMemoryError();
        //如果size+1 > Integer.MAX_VALUE - 8 直接返回Integer.MAX_VALUE 否则返回Integer.MAX_VALUE - 8
        return (minCapacity > MAX_ARRAY_SIZE) ?
            Integer.MAX_VALUE :
            MAX_ARRAY_SIZE;
    }

思考：

  1. 判断是否需要扩容，拆分成了3个方法，每个方法的职责很明确，可读性很高。
  2. 进行扩容时，数组的索引不能小于0 或者是大于Integer.MAX_VALUE，这是有防止数组下标溢出的意识。

5.2 ArrayList的缩容机制

   //对ArrayList的大小进行调整
   public void trimToSize() {
        //记录数组的修改次数
        modCount++;
        //如果元素数量小于数组的长度
        if (size < elementData.length) {
            //如果size==0 elementData指向空数组，否则elementData指向新创建的size大小的复制数组
            elementData = (size == 0)? EMPTY_ELEMENTDATA : Arrays.copyOf(elementData, size);
        }
    }

思考：这样会创建大小刚好用的数组

   //保证最小容量 
   public void ensureCapacity(int minCapacity) {
        //如果不是用的空参构造就赋值为0，否则就是用的空参构造，赋值为默认容量
        int minExpand = (elementData != DEFAULTCAPACITY_EMPTY_ELEMENTDATA) ? 0: DEFAULT_CAPACITY;
        //如果指定的参数 > 0或10
        if (minCapacity > minExpand) {
            //进行扩容
            ensureExplicitCapacity(minCapacity);
        }
    }

思考：可以理解为主动扩容

5.3 添加单个元素

  public boolean add(E e) {
        //判断是否需要扩容
        ensureCapacityInternal(size + 1);  // Increments modCount!!
        //直接在数组末尾添加元素
        elementData[size++] = e;
        return true;
    }

思考：

  1. 新增完成后,并没有进行任何的锁控制，而是直接在末尾添加元素，这种方式不是线程安全的。
  2. 新增时并没有对数据进行校验，所以是允许添加null值的。
  3. 这个方法的时间复杂度为O(1), 空间复杂度为O(1)。

5.3 添加单个元素到指定位置

 public void add(int index, E element) {
        //对索引进行检查
        rangeCheckForAdd(index);
        //判断是否需要扩容
        ensureCapacityInternal(size + 1); 
        //将index开始的所有元素往后移动一位
        System.arraycopy(elementData, index, elementData, index + 1,size - index);
        //将元素插入到index的位置
        elementData[index] = element;
        //数组大小+1
        size++;
    }

 private void rangeCheckForAdd(int index) {
        //如果传入的索引比数组元素数量还大，或者小于0
        if (index > size || index < 0)
            //抛出索引越界异常
            throw new IndexOutOfBoundsException(outOfBoundsMsg(index));
    }

 private String outOfBoundsMsg(int index) {
        //返回错误的索引信息
        return "Index: "+index+", Size: "+size;
    }

思考：一段重复使用的代码可以定义方法进行调用，减少劳动,这种方式添加元素的 : 最坏时间复杂度为O(n),最好为O(1),平均时间复杂度为O(n),空间复杂度O(1)。

5.4 添加多个元素

   //将集合对象的所以元素添加到ArrayList的末尾
   public boolean addAll(Collection<? extends E> c) {
        //将集合对象转化为对象数组
        Object[] a = c.toArray();
        //a数组的长度
        int numNew = a.length;
        //集合必须要容纳size + numNew个元素
        ensureCapacityInternal(size + numNew);
        //将a数组的所有元素添加到集合的末尾
        System.arraycopy(a, 0, elementData, size, numNew);
        //size=原来集合元素的数量+新增的数量
        size += numNew;
        return numNew != 0;
    }

    //往指定位置添加一个集合对象的所有元素
    public boolean addAll(int index, Collection<? extends E> c) {

        rangeCheckForAdd(index);

        Object[] a = c.toArray();
        int numNew = a.length;
        ensureCapacityInternal(size + numNew);  
        //需要移动多少个元素
        int numMoved = size - index;

        if (numMoved > 0)
        //让出要插入元素的位置    
        System.arraycopy(elementData, index, elementData, index + numNew,numMoved);
        //将要插入的元素插入
        System.arraycopy(a, 0, elementData, index, numNew);
        size += numNew;
        return numNew != 0;
    }

思考：这种方式可以将任意实现了Collection接口的对象的所有元素添加进来，可以避免多次扩容

5.5 移除单个元素

   //移除索引处的元素
   public E remove(int index) {
        rangeCheck(index);

        modCount++;
        E oldValue = elementData(index);
        //要移动的元素个数  
        int numMoved = size - index - 1;
        //如果需要移动
        if (numMoved > 0)
        //将index的后面的元素往前移动一位 ，这样只需要删除最后一个元素   
        System.arraycopy(elementData, index+1, elementData, index, numMoved);
        //只是将数组最后一个元素的引用取消了，对象并未置空，垃圾回收会做这件事情
        elementData[--size] = null; 
        //返回旧值
        return oldValue;
    }
    //移除给定对象的元素
    public boolean remove(Object o) {
        //如果是要删除为null的元素
        if (o == null) {
            //遍历整个集合找出元素进行删除
            for (int index = 0; index < size; index++)
                if (elementData[index] == null) {
                    fastRemove(index);
                    return true;
                }
        } else {
            //遍历整个集合找出元素进行删除
            for (int index = 0; index < size; index++)
                //对象之间的相等需要用equals进行判断
                if (o.equals(elementData[index])) {
                    fastRemove(index);
                    return true;
                }
        }
        return false;
    }
    //相当于移除索引处的元素
    private void fastRemove(int index) {
        modCount++;
        int numMoved = size - index - 1;
        if (numMoved > 0)
        System.arraycopy(elementData, index+1, elementData, index,numMoved);
        elementData[--size] = null;
    }

思考：使用索引删除的时间复杂度是：最坏时间复杂度是O(n),最好时间复杂度是O(1),平均时间复杂度为O(n)，使用对象删除的方式无论怎么样都是O(n)的时间复杂度，由此得出ArrayList的删除是比较低效的。

5.6 移除多个元素

    public boolean removeAll(Collection<?> c) {
        //判断对象是否为空
        Objects.requireNonNull(c);
        return batchRemove(c, false);
    }

    public static <T> T requireNonNull(T obj) {
        if (obj == null)
            throw new NullPointerException();
        return obj;
    }
    //批量删除元素
    private boolean batchRemove(Collection<?> c, boolean complement) {
        //用一个elementData指向本集合的elementData
        final Object[] elementData = this.elementData;
        // r用来遍历整个集合，w记录保留多少个元素
        int r = 0, w = 0;
        //删除成功后置于true
        boolean modified = false;  
        try {
            //遍历整个集合
            for (; r < size; r++)
                //如果没有找到对应元素
                if (c.contains(elementData[r]) == complement)
                    //按顺序保留，要删除的直接覆盖
                    elementData[w++] = elementData[r];
        } finally {
            //如果进行删除过程中，中途或者直接发生了错误
            if (r != size) {
                //直接保留未遍历的元素
                System.arraycopy(elementData, r, elementData, w,size - r);
                //要删除的元素=前面保留过的元素+已经保存的未删除元素，然后删除
                w += size - r;
            }
            //如果是要删除元素
            if (w != size) {
                // 将已经保留过的值删除
                for (int i = w; i < size; i++)
                    elementData[i] = null;
                //集合被修改的次数=当前的modCount+删除的元素个数
                modCount += size - w;
                //剩下的元素个数
                size = w;
                //删除成功
                modified = true;
            }
        }
        return modified;
    }
    //查看某个元素是否存在
    public boolean contains(Object o) {
        return indexOf(o) >= 0;
    }
    //返回元素所在的索引
    public int indexOf(Object o) {
        if (o == null) {
            for (int i = 0; i < size; i++)
                if (elementData[i]==null)
                    return i;
        } else {
            for (int i = 0; i < size; i++)
                if (o.equals(elementData[i]))
                    return i;
        }
        return -1;
    }

思考：与删除单个对象的实现思路差不多，这个方法还考虑了中途或者直接发生错误的情况。

5.9 迭代器

    //返回迭代器对象
    public Iterator<E> iterator() {
        return new Itr();
    }
    //实现迭代器
    private class Itr implements Iterator<E> {
        int cursor;       // 下一次返回的元素索引，默认从0开始  
        int lastRet = -1; // 遍历场景中，表示上一次遍历的索引，删除场景中，表示当前要删除的索引。
        //等于集合的modCount
        int expectedModCount = modCount; 
        //判断是否还有元素
        public boolean hasNext() {
            return cursor != size;
        }

        //遍历集合元素
        public E next() {
            //检查是否发生并发异常
            checkForComodification();
            int i = cursor;
            //如果i比size大，抛出没有这样的元素异常
            if (i >= size)
                throw new NoSuchElementException();
            //指向集合的elementData
            Object[] elementData = ArrayList.this.elementData;
            //如果i>=集合的长度，抛出发生并发异常
            if (i >= elementData.length)
                throw new ConcurrentModificationException();
            //下一次迭代的元素位置
            cursor = i + 1;
            //返回元素
            return (E) elementData[lastRet = i];
        }

        public void remove() {

            if (lastRet < 0)
                throw new IllegalStateException();

            checkForComodification();

            try {
                //调用自身的索引删除方法
                ArrayList.this.remove(lastRet);
                //等于当前的要删除的索引
                cursor = lastRet;
                //重新恢复判断状态，表示元素已经删除，防止不正确的使用：必需先调用next()一次方法才能使用一次remove方法
                //防止重复删除
                lastRet = -1;
                //调用了自身的remove方法 更新modCount
                expectedModCount = modCount;
            } catch (IndexOutOfBoundsException ex) {
                throw new ConcurrentModificationException();
            }
        }

        final void checkForComodification() {
            if (modCount != expectedModCount)
                throw new ConcurrentModificationException();
        }
